「一手ずつ、あなたの将棋を強くする。」をコンセプトにした、オンライン将棋教室の公式ホームページです。
HTML / CSS / JavaScript のみで構築されており、Bootstrap・Tailwind・React・Vue・jQuery等のライブラリやフレームワークは一切使用していません。

## サイト概要

- 対象: 将棋を学びたい初心者から有段者まで
- 目的: 無料体験レッスンへの申し込み（Googleフォームへ送客）
- デザイン: 白基調 × 紺色 × 金色のアクセント、余白を活かしたモダンなデザイン
- 技術: HTML5 / CSS3（CSS変数によるトークン管理） / Vanilla JavaScript（ES6, IIFE構成）

├── style.css                … スタイルシート（CSS変数で色・余白等を管理）

├── blog/

> **画像について**: `images/` 内の画像は、提供いただいたサンプル画像をもとに実写真スタイルの素材へ差し替え済みです。ご自身の教室写真・講師写真に差し替えたい場合は、下記「画像差し替え方法」の手順で上書きしてください。HTML・CSS・JavaScript側は、どんな画像に差し替えてもレイアウトが崩れないように実装されています。

## GitHub Pages公開方法

1. GitHubで新しいリポジトリを作成します（例: `shogi-school`）。
2. `shogi-school/` フォルダの中身一式（`index.html` や `images/` を含む全ファイル）をリポジトリのルートにアップロードします。
   - フォルダごとではなく、**中身をルート直下に配置**してください。
3. リポジトリの `Settings` → `Pages` を開きます。
4. `Source` を `Deploy from a branch` に設定し、対象ブランチ（例: `main`）と `/ (root)` を選択して保存します。
5. しばらく待つと `https://ユーザー名.github.io/リポジトリ名/` で公開されます。
6. `index.html` 内の `https://example.com/` となっている箇所（`canonical` / OGP / 構造化データ等）を、実際に公開するURLに置き換えてください。

すべて相対パス（`./` から始まるパス）で構成しているため、フォルダ名が変わってもリンク切れは発生しません。

## 画像差し替え方法

`images/` フォルダ内の画像は、すべて**差し替え前提のプレースホルダー**です。以下の手順で、実際の写真に差し替えてください。

### 手順

1. 差し替えたい画像と**同じファイル名・同じ拡張子**で、新しい画像を用意します。
   - 例: 講師写真を差し替える場合 → `profile.webp` という名前で画像を用意する
2. `images/` フォルダ内の既存ファイルを、用意した新しい画像で上書きします。
3. ファイル名を変える場合は、`index.html` 内の該当する `<img>` タグ・`<source srcset>` タグのパスも合わせて変更してください。
4. ブラウザでページを再読み込みし、表示を確認してください。

### プレースホルダー画像一覧と用途

| ファイル名 | 用途 | 推奨サイズ（幅×高さ） | 備考 |
|---|---|---|---|
| `images/hero.webp` | トップのヒーロー背景（将棋盤・駒の写真など） | 1920×1200程度 | 横長、容量250KB以内が目安 |
| `images/profile.webp` | 講師プロフィール写真 | 800×800程度（正方形） | 丸型にトリミング表示されます。容量100KB以内が目安 |
| `images/board.webp` | 将棋盤の予備素材 | 1200×900程度 | 現状は未使用です。盤のマス目の線がAI生成特有に歪んでいるため、使用する際はご注意ください |
| `images/about.webp` | レッスン内容セクションの写真／CTA（無料体験申し込み）セクションの背景写真（共通） | 1200×900程度 | 2箇所で同じ画像を使用しています |
| `images/logo.png` | ヘッダー・フッターのロゴマーク | 512×512程度（透過PNG推奨） | |
| `images/ogp.jpg` | SNSでシェアされた際に表示される画像 | 1200×630固定 | |
| `images/emblem.webp` | フッターに表示される装飾用エンブレム画像 | 500×500程度 | 白い背景のカード上に表示されます |
| `images/blog-start.webp` | ブログ記事用の写真（ペンと駒） | 1200×838程度 | |
| `favicon.ico` / `images/favicon.ico` | ブラウザタブに表示されるアイコン | 16〜64px相当のico形式 | |

### 画像形式・圧縮について

- 写真素材は **WebP形式** を推奨します（同画質でJPEGより軽量です）。
- 無料の画像変換ツールやオンラインコンバーターで、JPEG/PNGからWebPへ変換できます。
- 画像を大きくしすぎるとページの表示速度（LCP）が低下するため、上記の推奨サイズ・容量を目安に圧縮してください。

## Googleフォーム変更方法

現在は、実際のGoogleフォームのURLが設定済みです。フォームを作り直すなどしてURLを変更したい場合は、`index.html` 内を `docs.google.com/forms` で検索し、以下の箇所のURLを新しいものに差し替えてください。

```html
<a href="https://docs.google.com/forms/d/e/xxxxxxxx/viewform" class="btn btn--primary btn--large" target="_blank" rel="noopener noreferrer">
  無料体験を申し込む
</a>
```

   このリンクは、CTAセクション（ページ下部の「まずは無料体験から始めてみませんか？」）内にあります。ヘッダーやヒーローの「無料体験を申し込む」ボタンは同じページ内の `#contact` セクションへスクロールする仕様のため、変更は上記1箇所のみで反映されます。

## 価格変更方法

料金は `index.html` 内の `id="pricing"` セクションに記載されています。

無料体験カードは、金額ではなく「まずは費用をかけずにお試しいただけます」という案内文で表現しています。文言を変更したい場合は、以下の箇所を編集してください。

```html
<p class="pricing__trial-copy">まずは費用をかけずにお試しいただけます</p>
```

60分レッスンの金額を変更したい場合は、以下の箇所を編集してください。

```html
<p class="pricing__price"><span>¥</span><span data-price="single">2,500</span></p>
```

`data-price` 属性は将来的にJavaScriptから価格を一括管理したい場合の目印として付与しています。金額の数字部分のみを書き換えれば表示に反映されます。プランを追加する場合は、既存の `.pricing__card` を1つ複製して `.pricing` 内に追加してください。

## ブログ記事の追加方法

`blog/` フォルダに、ブログ一覧ページ（`blog/index.html`）と、サンプル記事（`blog/sample-post.html`）を用意しています。サンプル記事は、そのまま新しい記事を書くときのテンプレートとしても使えます。

### 新しい記事を書く手順

1. `blog/sample-post.html` をコピーして、新しいファイル名を付けます（例: `blog/2026-08-benkyohou.html`）。日付や内容が分かるファイル名にしておくと管理しやすいです。
2. コピーしたファイルを開き、以下を書き換えます。
   - `<title>` タグ
   - `<meta name="description">`
   - `<link rel="canonical">` のURL
   - ページ内の `BreadcrumbList` と `BlogPosting` の構造化データ（タイトル・日付・URL）
   - `<article>` 内の見出しと本文
   - 公開する記事は `<meta name="robots">` を `index, follow` に変更してください（サンプル記事は `noindex` にしているため検索結果に表示されません）。
3. `blog/index.html` を開き、`<div class="blog-grid">` 内に新しい記事へのリンクカード（`<article class="blog-card">...</article>`）を1つ追加します。既存のカードをコピーして、リンク先・タイトル・日付・抜粋文を書き換えるのが簡単です。
4. ルートの `sitemap.xml` に、公開した記事のURLを追加します。

### 記事の画像について

記事のサムネイルには `images/` フォルダ内の画像を使うか、新しい画像を `images/` フォルダに追加して使うこともできます。画像の差し替え方法は「画像差し替え方法」の章を参照してください。

## SEO変更方法

`index.html` の `<head>` 内に、以下のSEO関連タグがまとまっています。

- `<title>` … 32文字以内を目安にしたページタイトル
- `<meta name="description">` … 120文字程度の説明文
- `<meta name="keywords">` … 想定キーワード
- `<link rel="canonical">` … 正規URL（公開URLに変更してください）
- `<meta property="og:*">` / `<meta name="twitter:*">` … SNSシェア時の表示情報
- `<script type="application/ld+json">` … 構造化データ（LocalBusiness / Person / WebSite / Organization / BreadcrumbList / FAQPage）

公開URLが決まったら、`https://example.com/` という記述をすべて実際のURLに置換してください（`canonical` タグ、OGPタグ、構造化データ内の `url` / `image`、`robots.txt` と `sitemap.xml` 内のURLが対象です）。

## 動作確認したブレークポイント

- 320px（小型スマートフォン）
- 480px
- 768px（タブレット縦）
- 1024px（タブレット横・小型ノートPC）
- 1280px以上（PC）

いずれの画面幅でもレイアウト崩れが起きないよう、CSS Grid・Flexboxと `clamp()` を用いたフォントサイズ調整で対応しています。

## 特定商取引法に基づく表記について

`tokushoho.html` に、特定商取引法に基づく表記のページを用意しています。販売事業者名・運営統括責任者には、すでにお名前を記載済みです。公開前に、以下の点をご確認ください。

- 住所・電話番号は「ご請求をいただいた場合には、遅滞なく開示いたします」という表記のままにしておくことで、常時公開せずに済ませることができます。
- 支払い方法・キャンセルポリシーなど、実際の運用と異なる場合は書き換えてください。

このページが実際にご自身の事業に対して法的に必要かどうかは、税理士など専門家にご確認いただくことをおすすめします。
